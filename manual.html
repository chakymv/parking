Manual del Backend ParkingLot

¿Qué es este proyecto?

Este backend es una API RESTful desarrollada en Node.js y Express para la gestión de un sistema de parqueadero inteligente. Utiliza Supabase/PostgreSQL como base de datos y está estructurada para ser robusta, escalable y fácil de mantener.

  • Todos los modelos de datos están desacoplados y centralizados en la carpeta model/.
  • Los endpoints REST están en api/ y cada uno usa su modelo correspondiente.
  • La conexión a la base de datos es unificada y segura mediante supabaseClient.js y variables de entorno.
  • Incluye pruebas automatizadas para todos los endpoints principales usando Jest y Supertest.

El código está pensado para ser claro, validando datos y devolviendo errores útiles, facilitando la integración con frontend o apps móviles.

1. Requisitos Previos

  • Node.js 18.x o superior
  • Acceso a una base de datos PostgreSQL (Supabase)
  • Git (opcional, para clonar el repositorio)

2. Instalación de Paquetes

  1. Clona o descarga el repositorio en tu máquina local.
  2. Abre una terminal en la carpeta raíz del proyecto.
  3. Ejecuta:
    npm install

2.1. Dependencias y Librerías Instaladas

  • @supabase/supabase-js: Cliente oficial para interactuar con Supabase/PostgreSQL.
  • express: Framework para crear el servidor y endpoints REST.
  • dotenv: Carga variables de entorno desde el archivo .env.
  • body-parser: Middleware para parsear el body de las peticiones (integrado en Express 4.16+).
  • supertest: Utilidad para pruebas HTTP automatizadas (devDependency).
  • serverless-http: Adaptador para desplegar Express en entornos serverless (opcional).
  • mysql2 y postgres: Drivers de bases de datos (no usados en la versión Supabase, pero presentes).

Para instalar todas las dependencias ejecuta: 

npm install

3. Configuración de Variables de Entorno

  1. Crea un archivo .env en la raíz del proyecto (si no existe).
  2. Agrega tus credenciales de Supabase/PostgreSQL, por ejemplo:
    SUPABASE_URL=tu_url_supabase
SUPABASE_KEY=tu_api_key

4. Estructura del Proyecto

  • api/: Endpoints REST (usuarios, vehiculos, celdas, etc.)
  • model/: Modelos de datos y lógica de negocio
  • test/: Pruebas automatizadas con Jest y Supertest
  • supabaseClient.js: Configuración de conexión a Supabase
  • app.js y server.js: Arranque del servidor Express

5. Ejecución del Servidor

  1. En la terminal, ejecuta:
    node server.js
  2. El backend estará disponible en http://localhost:3000 (o el puerto configurado).

6. Ejecución de Pruebas Automatizadas

  1. Para ejecutar todas las pruebas:
    npx jest --runInBand
  2. Para ejecutar una prueba específica:
    npx jest --runInBand test/PruebasCelda.test.js
  3. Las pruebas cubren todos los endpoints principales (usuarios, vehículos, celdas, historial de parqueo, acceso/salidas).

7. Buenas Prácticas

  • Todos los modelos usan conexión unificada a Supabase y nombres de columnas en minúsculas.
  • Los endpoints validan campos obligatorios y devuelven errores claros.
  • Las pruebas automatizadas aseguran la integridad de la API.

Para dudas o soporte, contacta al equipo de desarrollo o revisa la documentación interna del proyecto.
